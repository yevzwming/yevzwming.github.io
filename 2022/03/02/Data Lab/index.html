CSAPP: Data Lab

Preparations: 环境搭建

本实验源于cmu的csapp课程网站，实验材料在课程实验官网上下载。

该实验主要考察学生对位级运算、无符号数编码、补码以及浮点数编码的掌握程度。

首先我们搭配实验环境，我们需要一个linux环境，比较适用的就是Ubuntu20.04系统。我为了图方便，直接在window11上安装了一个linux子系统，即wsl2。具体操作可以参考此处（Ubuntu建议直接microsoft商店下载）

首先我们按照实验手册，打开终端。输入以下命令：

> tar xvf datalab-handout.tar

于是我们可以看见bits.c,我们所有的操作都是在bits.c中完成。

在实验过程中，我们可以check我们的得分，通过终端输入命令来实现：

> make
> ./btest

btest.c是一个测试文件，于是我们可以开始我们的上分之旅啦！

Lab: 上分之旅

我们开始一关关的打过去。

bitXor

/*
 * bitXor - x^y using only ~ and &
 *   Example: bitXor(4, 5) = 1
 *   Legal ops: ~ &
 *   Max ops: 14
 *   Rating: 1
 */
int bitXor(int x, int y)
{
  return ~(~(~x & y) & ~(x & ~y));
}

要求我们实现异或，只能使用~和&两个符号。异或是什么？就是一样就是0，不一样就是1，我们用“或”来搞他的不一样，但是|我们用不起来，所以用 ~和&来替代（取反），于是有了如上的简单式子。

很trivial奥。

tmin

/*
 * tmin - return minimum two's complement integer
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 4
 *   Rating: 1
 */
int tmin(void)
{
  return 1 << 31;
}

这题不用看了，取最小整数，直接左移31位结束。

isTmax

/*
 * isTmax - returns 1 if x is the maximum, two's complement number,
 *     and 0 otherwise
 *   Legal ops: ! ~ & ^ | +
 *   Max ops: 10
 *   Rating: 1
 */
int isTmax(int x)
{
  int y = x + 1;
  return (!((~y + 1) ^ y)) & (!!y);
}

我们思考Tmax有着什么样的性质：Tmax+1=Tmin！不得了，Tmin我们可是太熟悉了，因为我们有Tmin和-Tmin是同一个数！但是同样具有这个性质的还有0，于是我们用!!y来排除0的情况。判断是不是同一个数，我们直接用异或符号^来判断，而取负即是~y+1。Over ~

allOddBits

/*
 * allOddBits - return 1 if all odd-numbered bits in word set to 1
 *   where bits are numbered from 0 (least significant) to 31 (most significant)
 *   Examples allOddBits(0xFFFFFFFD) = 0, allOddBits(0xAAAAAAAA) = 1
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 12
 *   Rating: 2
 */
int allOddBits(int x)
{
  int m = 0xAA + (0xAA << 8);
  m = m + (m << 16);
  int mm = m & x;
  return !(mm ^ m);
}

这题要求我们判断一个数它的奇数位是不是全1。我觉得可以采用掩码思想，我们用一个m,首先0xAA是10101010嘛，于是用移位操作覆盖整个32位的奇数位。然后我们用这个掩码mm与待判断的数x的进行&操作。如果m它bingo呢。就会和mm一样，最后就会返回0，！一下就是1。于是用异或操作即可。

negate

/*
 * negate - return -x
 *   Example: negate(1) = -1.
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 5
 *   Rating: 2
 */
int negate(int x)
{
  return ~x + 1;
}

不明白这为啥不是第一题，没啥好说的，负数就是取反加一。

isAsciiDigit

/*
 * isAsciiDigit - return 1 if 0x30 <= x <= 0x39 (ASCII codes for characters '0' to '9')
 *   Example: isAsciiDigit(0x35) = 1.
 *            isAsciiDigit(0x3a) = 0.
 *            isAsciiDigit(0x05) = 0.
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 15
 *   Rating: 3
 */
int isAsciiDigit(int x)
{
  int min = x + ~0x30 + 1;
  int max = x + ~0x39 + 1;
  return ((!max) | max >> 31) & !(min >> 31);
}

这题的本质呢，是一个比较大小。比较大小是什么呢，当然不能if，所以我们作差看符号。减去下限就是x + ~0x30 + 1，上限同理。需要同时满足，所以结构我们用&。左边应该是正的，符号位为零，所以右移31位取反；右边应该是负的或者是0，所以有个|符号，大概差不多道理便可以得到上面那个式子啦。

conditional

/*
 * conditional - same as x ? y : z
 *   Example: conditional(2,4,5) = 4
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 16
 *   Rating: 3
 */
int conditional(int x, int y, int z)
{
  int t = (!x) << 31 >> 31;
  return (~t & y) | (t & z);
}

这题要实现一个条件判断，也就是说看这个x是否为零。是否为零，我们用！来判断更加适合，于是用一个t来移位骚操作一番来搞到这么个效果：非零即零，零即-1，即全是1。所以我们要取一个数，就把另外一个数与0&一下，搭配一下我们得到上面这个选择结构。

isLessOrEqual

/*
 * isLessOrEqual - if x <= y  then return 1, else return 0
 *   Example: isLessOrEqual(4,5) = 1.
 *   Legal ops: ! ~ & ^ | + << >>
 *   Max ops: 24
 *   Rating: 3
 */
int isLessOrEqual(int x, int y)
{
  int m = ~x + 1 + y;
  int bit = 0x1 & ((x ^ y) >> 31); //相同为0，不同为1
  // 0
  int s = m >> 31;
  // 1:判断位数
  return ((!s) & (!bit)) | (bit & (x >> 31));
}

比较大小？首先联想到作差法。我们用大的减小的，是不是模仿上面那个ascll码就出来了呢？NO!你考虑溢出了吗？？在一个正的和一个负的的情况下，直接相减可能会爆掉！于是我们要分开来讨论。怎么讨论呢？我想到一个很牛逼的想法，对两个数的符号位取异或！如果两个符号不同，就得bit为1并且x是负的（即x的符号位为1）；如果两个符号相同，我们就可以放心得减了，直接通过差判断，中间用|即可。

LogicalNeg

/*
 * logicalNeg - implement the ! operator, using all of
 *              the legal operators except !
 *   Examples: logicalNeg(3) = 0, logicalNeg(0) = 1
 *   Legal ops: ~ & ^ | + << >>
 *   Max ops: 12
 *   Rating: 4
 */
int logicalNeg(int x)
{
  //符号位相同的只有两个数，排除去另一个，用符号位
  int bit = (x >> 31) & 0x1;
  int bbit = ((~x + 1) >> 31) & 0x1;
  int isZero = bit ^ bbit;
  return !isZero & !bit;
}

同样，我们要判断一个数，就是要看这个数有没有什么特征。在这里我们要判断0，0有什么特征？0和0的负数符号位相同！符号位相同的只有两个数，我们排除去另一个即Tmin。我们只要对0进行一个简单判断就行了，就直接看原数的符号位bit。这不轻轻松松嘛！

howManyBits

/* howManyBits - return the minimum number of bits required to represent x in
 *             two's complement
 *  Examples: howManyBits(12) = 5
 *            howManyBits(298) = 10
 *            howManyBits(-5) = 4
 *            howManyBits(0)  = 1
 *            howManyBits(-1) = 1
 *            howManyBits(0x80000000) = 32
 *  Legal ops: ! ~ & ^ | + << >>
 *  Max ops: 90
 *  Rating: 4
 */
int howManyBits(int x)
{
  int flag = x >> 31;
  x = (flag & (~x)) | ((~flag) & x);
  int b16 = !!(x >> 16) << 4;
  x >>= b16;
  int b8 = !!(x >> 8) << 3;
  x >>= b8;
  int b4 = !!(x >> 4) << 2;
  x >>= b4;
  int b2 = !!(x >> 2) << 1;
  x >>= b2;
  int b1 = !!(x >> 1);
  x >>= b1;
  int b0 = x;
  return b0 + b1 + b2 + b4 + b8 + b16 + 1;
}

这题要数一个数最少要多少位可以表示其二进制。

摊牌了，这题我不会，上网借鉴别人思路搞出来的。在此简单说下这个程序大概什么结构吧。我们简单思考一下可以发现，负数取反和正数是一样子处理的：正数是取最高的1，负数就是最高的0。于是我们骚一下，统一变成一种形式，用全一的数进行&。相当于做个标本。

然后呢我们从高位开始判断起，高16位是否有1，如果有，好了，16位肯定是要的了，我们继续分析；怎么看呢？就是右移16位看是否为零，用！！来做，然后再左移4位，就能实现要么0要么16的效果了。下面每次8，4，2，1同样的操作下去，最后return一个和。

这题真的好难想。

floatScale2

unsigned floatScale2(unsigned uf)
{
  unsigned exp = (uf & 0x7f800000) >> 23;
  unsigned sign = uf >> 31 & 0x1;
  unsigned frac = uf & 0x7fffff;
  if (exp == 0xFF)
    return uf;
  else if (exp == 0)
    return (sign << 31) + (exp << 23) + (frac << 1);
  else
    return (sign << 31) + ((++exp) << 23) + frac;
}

终于进入浮点数阶段了。这题我们要求一个float乘以2。

一般基本想法都是把float的三个部分单独取出来表示，分别是sign, exp和frac。

首先我们判断是不是NaN或者Inf，我们直接看指数位，满的话直接return uf。

剩下来的我们要分规格化数还是非规格化数来判断。规格化数直接指数加一，非规格化数需要把frac部分移位，即乘以2。于是我就在想啊，如果frac快要满了就差一点点就规格化了呢？你肯定也在想。注意！这里的数都是32位的！你frac进上去并不会溢出，只是进位到了exp上，而浮点数的表示是连续的！

于是好像就全部解决了。

floatFloat2Int

/*
 * floatFloat2Int - Return bit-level equivalent of expression (int) f
 *   for floating point argument f.
 *   Argument is passed as unsigned int, but
 *   it is to be interpreted as the bit-level representation of a
 *   single-precision floating point value.
 *   Anything out of range (including NaN and infinity) should return
 *   0x80000000u.
 *   Legal ops: Any integer/unsigned operations incl. ||, &&. also if, while
 *   Max ops: 30
 *   Rating: 4
 */
int floatFloat2Int(unsigned uf)
{
  unsigned exp = (uf & 0x7f800000) >> 23;
  unsigned sign = (uf >> 31) & 0x1;
  unsigned frac = uf & 0x7fffff;
  int e = exp - 127;
  if (e < 0)
    return 0;
  else if (e >= 31)
    return 0x80000000u;
  else
  {
    frac = frac | 1 << 23;
    if (e < 23)
      frac >>= (23 - e);
    else
      frac <<= (e - 23);
  }
  if (sign)
    return -frac;
  else
    return frac;
}

首先我们还是老样子，把三个部分都表示出来。首先把bias减掉得出最后乘的e。

小数的话直接return 0， 当$e\geq 31$时，超过int啦，爆掉了，所以也先乖乖return吧。

剩下的情况中，我们把frac先全提成整数。下面还是要分类讨论！因为e可能小于23，就会有小数，而我们需要int，所以要舍入。稍微转一下脑子，移移位我们可以把这题搞定了。

我可不会说这题我也是借鉴代码的。

floatPower2

/*
 * floatPower2 - Return bit-level equivalent of the expression 2.0^x
 *   (2.0 raised to the power x) for any 32-bit integer x.
 *
 *   The unsigned value that is returned should have the identical bit
 *   representation as the single-precision floating-point number 2.0^x.
 *   If the result is too small to be represented as a denorm, return
 *   0. If too large, return +INF.
 *
 *   Legal ops: Any integer/unsigned operations incl. ||, &&. Also if, while
 *   Max ops: 30
 *   Rating: 4
 */
unsigned floatPower2(int x)
{
  if (x > 127)
    return 0xff << 23;
  else if (x < -148)
    return 0;
  else if (x >= -126)
  {
    int exp = x + 127;
    return exp << 23;
  }
  else
    return 1 << (x + 148);
}

这题其实没啥了，就是计算各种情况的边界。首先x过大，直接return inf了；然后很小的话咱也直接return 0；剩下来的如果x小，但小的没那么离谱，咱就直接移+148位，因为还没规格化呢；最后就直接exp的移位了。这题反而感觉是浮点数题里比较简单的。
